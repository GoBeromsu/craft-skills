# Changelog

- 2026-06-13 — no documentation ontology existed → established the `docs/` ontology skill: six artifact types (research/references/spec/plan/decision/rule), the research→decision→plan pipeline, plan-vs-ADR boundary, lifecycle rules, and eight canonical [templates](templates/).
- 2026-06-25 — supersede-chains made ADRs hard to follow across edits → switched the ADR model to self-complete in-place edits with an in-doc `## Changelog`, dropping supersede/coverage-matrix/retired-source tracking from guidance and the [decision template](templates/adr.md).
- 2026-06-25 — our ADR guidance lacked why-not-what framing and when-to-write criteria → absorbed them from [addyosmani documentation-and-adrs](https://github.com/addyosmani/agent-skills/blob/main/skills/documentation-and-adrs/SKILL.md) (common-rationalizations table, gotcha→ADR reference), keeping the self-complete + `## Changelog` model.
- 2026-06-25 — same-decision ADR changes still read as needing a supersede → documented the supersede-decomposition model (in-place edit + `## Changelog` for same-decision changes; References/Refines links between atomic ADRs) and removed `superseded-by` from the decision template.
- 2026-06-30 — v1.1.0: `documents` had grown into one fat SKILL.md → split into a waypoint + nested sub-recipes ([adr](references/adr.md), [readme](references/readme.md), api-docs, [changelog](references/changelog.md), [inline-comments](references/inline-comments.md)), extracting ADR depth from the waypoint and absorbing README/API-docs/changelog/comment formats from [addyosmani documentation-and-adrs](https://github.com/addyosmani/agent-skills/blob/main/skills/documentation-and-adrs/SKILL.md). Additive (MINOR). Provenance: ADR model + README/API-docs/changelog/comment formats derived from the `documentation-and-adrs` SSOT @ addyosmani/agent-skills (see [skills/PROVENANCE.md](../PROVENANCE.md)).
- 2026-07-05 — v1.2.0: UI work needs a single design-system source of truth before code → add [design/](references/design.md) sub-recipe (7-section design.md contract, states-before-code primitive rule, same-commit + staleness lifecycle). Provenance: design.md section contract with states-before-code adapted from DESIGN.md convention @ code-yeongyu/lazycodex.
- 2026-07-05 — patch: [design/SKILL.md](references/design.md)'s staleness-audit command used a GNU-only `sed 's/\.[jt]sx\?$//'` BRE extension that BSD/macOS sed silently no-ops on, flagging every already-documented primitive as a false-positive orphan → switched to `sed -E 's/\.[jt]sx?$//'` (POSIX extended regex, portable across BSD and GNU sed).
- 2026-07-05 — v2.0.0: the `api-docs` sub-recipe (JSDoc/docstring + OpenAPI) is removed — API-surface documentation is a code-domain concern, not part of the docs/ ontology; consumers recover the recipe from git history (pre-2.0.0 commits). BREAKING (MAJOR).
