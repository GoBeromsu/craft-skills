# Changelog

- 2026-07-05 — v1.0.0: no standing discipline existed for when restructuring is warranted, how to do it safely on untested legacy code, or which smell maps to which mechanical fix → initial authoring; SKILL.md encodes when-to-refactor triggers (rule of three, preparatory/comprehension refactoring, boy-scout scope boundary), a legacy/untested-code characterization-test protocol, and a refactor-commits-separate-from-behavior-commits law with a git detection command; `references/code-smells.md` catalogs 17 smells across size/duplication/coupling/abstraction/change/comments with a threshold-bearing detect command per smell; `references/catalog.md` gives 12 mechanical moves with py+ts examples; `scripts/detect-smells.sh` runs the greppable subset in one pass. Provenance: smell-detection catalog format adapted from code-smells @ code-yeongyu/lazycodex; move catalog adapted from Martin Fowler, Refactoring (2nd ed.).
