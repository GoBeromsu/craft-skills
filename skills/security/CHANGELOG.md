# Changelog

- 2026-07-05 — v1.0.0: no cross-domain defensive security recipe existed for triaging web, API, and LLM vulnerabilities alongside secrets and dependency hygiene → adds a threat-model-first workflow (trust-boundary mapping, a three-tier action boundary, and a reachability × severity × fix-availability triage tree) that routes to [references/web.md](references/web.md), [references/api.md](references/api.md), [references/llm.md](references/llm.md), and [references/secrets-supply-chain.md](references/secrets-supply-chain.md), each shipping a copy-pasteable detection command per vulnerability class. Provenance: threat-model-first workflow and dependency-audit triage adapted from addyosmani/agent-skills @ github.com/addyosmani/agent-skills; LLM hardening rules adapted from OWASP Top 10 for LLM Applications.
- 2026-07-05 — patch: `allowed-tools` was missing `Write, Edit` even though the "Always do" action-boundary tier commits to applying source-level fixes and adding/updating tests, actions those tools require → aligned to `[Read, Write, Edit, Bash, Grep, Glob]`, matching the rest of the batch. Also expanded the undefined `RAG` acronym inline at its first use (SKILL.md's PHASE 0 routing table: "retrieval-augmented generation (RAG)") and in `references/llm.md`'s "RAG isolation" heading (now "RAG (Retrieval-Augmented Generation) isolation"), matching the package's existing acronym-expansion convention (XSS, CSP, CSRF, IDOR, SSRF).
