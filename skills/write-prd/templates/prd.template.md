## 1. One Pager

> 본격적인 PRD 작성 전에 작성하는 1 페이지 요약입니다.
> 의사결정자와 팀원이 제품 방향을 빠르게 이해하고, 우선순위를 논의하기 위한 섹션입니다.

### 1.1 Overview

<이 프로젝트가 어떤 제품/기능에 관한 것인지 설명합니다.>

#### 무엇을 채워야 하나요?

* 이 기능이 무엇인지
* 왜 중요한지
* 팀이 반드시 알아야 하는 핵심 맥락
* 이번 작업이 제품 전체에서 어떤 위치를 가지는지

#### 작성 예시

```text
이 프로젝트는 요양원 CCTV 영상을 기반으로 입소자의 낙상 위험 행동을 감지하고, 담당자에게 실시간 알림을 제공하는 기능입니다. 야간이나 교대 시간처럼 사람이 모든 병실을 동시에 확인하기 어려운 상황에서 위험 징후를 빠르게 인지하도록 돕는 것이 목적입니다.
```

### 1.2 Problem

<현재 상황과 고객의 Pain Point 를 설명합니다.>

#### 무엇을 채워야 하나요?

* 현재 사용자가 겪는 문제
* 이 문제가 왜 중요한지
* 기존 방식의 한계
* 이 문제가 비즈니스에 주는 영향
* 해결하지 않았을 때 발생하는 비용이나 리스크

#### 작성 예시

```text
현재 요양원 근무자는 야간과 교대 시간에 모든 병실의 상태를 실시간으로 확인하기 어렵습니다. 특히 침대 이탈, 휠체어 기립, 야간 배회 같은 낙상 전조 행동은 짧은 시간 안에 발생하므로 사람이 직접 CCTV를 계속 확인하는 방식만으로는 놓칠 가능성이 큽니다.
```
