# Changelog

- 2026-06-17 — initial authoring; correctness-first, type-strict programming skill for Python and TypeScript. Index SKILL.md routes through a PHASE 0 language gate to `references/python.md` and `references/typescript.md` (progressive disclosure). Establishes the ordering correctness > maintainability > brevity, the six-axiom shared philosophy (proof-system types, parse-don't-validate, branded primitives, exhaustive matching, boundary-only validation, TDD), the 250 pure LOC ceiling, the red-green-refactor + fakes-over-mocks test discipline, and a 9-point post-write review loop. Provenance: the type-strict iron list, 250 LOC ceiling, and post-write loop are adapted from code-yeongyu/lazycodex's `programming` skill; the "write only what the task needs" ladder, the native-platform rung (browser/CSS/DB-first), and the `craft:` deliberate-shortcut annotation (ceiling + upgrade path) are adapted from DietrichGebert/ponytail (MIT) including its open rung-3 web-guidance PR. Both sources are dissolved into present-tense recipe; brevity is demoted to a byproduct gated by correctness rather than ponytail's brevity-first stance, and Rust/Go are dropped to scope Python + TypeScript only.
- 2026-06-17 — v1.1.0: add `references/workflow.md` (always loaded via PHASE 0) encoding the task lifecycle — understand-before-edit, right-instrument tool use (symbol-aware rename / find-references, structural search over blind text replace), blast-radius callsite/test/doc updates, decompose-and-delegate without silently shrinking scope, the completion contract (no fabrication, no stub-as-feature, no green-by-suppression, claims match what was run), behavior-focused verification, and repo safety; add a compact "Working discipline" invariant block to `SKILL.md`. Provenance: this procedural layer is distilled from the gajae-code agent operating discipline and ponytail's lazy-but-not-negligent stance, generalized to be tool-agnostic for less-capable agents.
